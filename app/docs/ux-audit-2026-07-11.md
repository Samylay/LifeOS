# LifeOS UX/UI audit — 2026-07-11

Scope: all 11 surfaces (Today `/`, `/decide`, `/projects`, `/goals`, `/content`,
`/workouts`, `/knowledge`, `/pager`, `/status`, `/settings`, `/prime`) plus the
shell (sidebar, bottom nav, top bar, chat panel, toasts). Audited against the
9-surface IA, the interaction-craft doctrine (CLAUDE.md), WCAG 2.1 AA basics,
and Samy's decision-usefulness bar. Method: route smoke (all 200; `/brief`
307→`/` as designed), full component read, doctrine greps.

## What already holds (don't regress)

- IA is coherent: no orphan pages — every surface is reachable from the
  sidebar; `/prime` is intentionally reached from the Today quick-loop card
  rather than nav; `/brief` correctly redirects into Today.
- Motion doctrine passes its own greps: no `transition-all`, no animated
  layout properties, durations ≤500ms, custom easing tokens in use,
  `prefers-reduced-motion` block present, press feedback on nearly every
  actionable element, optimistic habit ticks and deck commits.
- `focus-visible` outline is defined globally; `<main>` landmark exists;
  confirm-dialog guards destructive deletes; empty states on decide decks are
  written as guidance ("new captures get studied nightly at 00:30"), which is
  the right bar for the whole app.

## Findings (ranked)

### HIGH

**H1 — `/decide` is absent from the mobile bottom nav.**
`components/bottom-nav.tsx` tabs: Home, Pager, Content, Projects, More.
The decision deck is Samy's newest daily loop, explicitly built for the phone,
yet on the phone it takes 2 taps through the More sheet. Fix: add a Decide tab
(6 items × 52px min fits ≥360px viewports). **FIXED this audit.**

**H2 — Pinch zoom is disabled app-wide** (`layout.tsx` viewport:
`maximumScale: 1, userScalable: false`). WCAG 1.4.4 violation; small-text
surfaces (26 uses of `text-[10px]`, dense status/containers rows) cannot be
zoomed. The block almost certainly exists to stop iOS auto-zoom on focused
inputs, but the correct fix is 16px form-control font on small screens, then
re-enable scaling. **FIXED this audit** (zoom enabled + `@media (max-width:
640px)` 16px input/select/textarea rule in `globals.css`).

**H3 — Undo toasts can sit under / collide with the bottom nav on mobile.**
Sonner 1.x hardcodes `--mobile-offset: 16px`; the fixed bottom nav is 64px +
safe-area. The undo toast is the safety net for the swipe deck — it must be
fully visible and tappable. **FIXED this audit** (CSS override lifting
`--mobile-offset` above the nav on <1024px).

### MEDIUM

**M1 — Icon-only buttons without an accessible name** (no `aria-label`, no
text). 12 flagged by the sweep; 5 were false positives (labels arrive via JSX
expressions: prime toggle, confirm-dialog pair, brief send/file). The 7 real
ones — status refresh, projects review-dismiss, chat clear/paste/send,
task-list status cycle, decide stop-recording — violated WCAG 4.1.2.
**FIXED this audit** (aria-labels added; status refresh also gained the
standard press feedback it was missing).

**M2 — Initial-load blankness / pop-in.** Today's brief section renders
nothing until `/api/brief-json` resolves, then the whole card column pops in
(layout shift on the home screen); `/status` shows only the header until the
first 8s poll returns; `/workouts` similar. Doctrine calls for skeletons on
genuinely slow loads. Queued as **T40**.

**M3 — Deck action buttons are ~36px tall** (`card-stack.tsx` `py-2`) — below
the 44px touch minimum on the app's most touch-centric surface; same for the
deck tab switcher (`py-1.5`). Queued as **T41**.

**M4 — `/decide` has no keyboard path.** Desktop users can click the buttons,
but arrow-key swipe (←/→ = the two swipe verdicts, with the same undo toast)
is the expected idiom for card decks and costs little. Queued as **T41**.

**M5 — Every page is titled "LifeOS".** Client-component pages can't export
`metadata`; a title template + per-route segment config (or a tiny
`useEffect(() => { document.title = … })` hook in each page) fixes history /
tab discrimination. WCAG 2.4.2 (minor here). Queued as **T42**.

### LOW

**L1 — `<main>` bottom padding ignores the safe-area inset** (`app-shell.tsx`
fixed `paddingBottom: 80`; nav itself already adds `env(safe-area-inset-bottom)`,
so on gesture-nav phones the last card can hide under the nav). **FIXED this
audit** (`calc(80px + env(safe-area-inset-bottom))`).

**L2 — Sidebar overlay dismiss button is a bare `<div onClick>`** (mobile
scrim) — not keyboard-reachable; Escape doesn't close the mobile sidebar.
Queued inside T41's interaction pass.

**L3 — `text-[10px]` (26 uses) for badges/mono annotations.** Acceptable as
badges, but several carry real information (container names on /status,
hook-strength numbers on /content). With zoom re-enabled (H2) this drops to
cosmetic; no action queued.

**L4 — Voice recording state is visual-only** (pulsing dot). A `role="status"`
live region announcing recording/thinking would make the flow non-visual-safe.
Folded into T41.

## Fixed in this audit (commits on main)

1. Decide tab in bottom nav (H1).
2. Pinch-zoom re-enabled + 16px mobile form controls (H2).
3. Mobile toast offset above bottom nav (H3).
4. aria-labels on the 7 unlabeled icon-only buttons + press feedback on status refresh (M1).
5. Safe-area-aware main padding (L1).
